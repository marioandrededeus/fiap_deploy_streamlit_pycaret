Deploy de app na cloud Streamlit

1) Criar conta github e streamlit (se aplicável)

GITHUB
2) Criar Personal acess token (se aplicável)
	2.1) clicar na imagem do perfil (canto sup direito)
	2.2) Settings
	2.3) Developer settings
	2.4) Personal acess tokens
	2.5) Tokens (classic)
	2.6) Generate new token (copiar e colar token em arquivo txt) 

3) Criar repositorio publico no Github
4) Copiar link https
5) no terminal, "cd" até a pasta desejada
6) git init -b main
7) git config --global user.email "email git"
8) git config --global user.name "user git"
9) git config --global credential.helper cache (para não solicitar token no proximo acesso)

10) git remote add origin https://<access_token>@github.com/user/repo

11) git remote -v   (opcional para confirmar se o origin foi salvo corretamente)

12) git add . ou git add [nome_arquivo]
13) git status
14) git commit -m "My first commit"
15) git push --set-upstream origin main
	15.1) git push --set-upstream --force origin main    
		Opcional para forçar o push. 
		Ela irá ignorar os commits do branch master do GitHub e o sobrescreverá com o seu branch local

EXTRA

GIT BRANCHES
1. Criar um novo branch:
git checkout -b novo_branch

2. Um branche apos criado ainda não está no servidor. Para publicá-lo:
git push novo_branch

3. Alterar entre branches:
git checkout novo_branch

4. Merge entre branches:
git merge master

5. Remover branch:
git branch -d novo_branch

FIZ BESTEIRA, E AGORA?
Digamos que você alterou alguma coisa localmente, se arrependeu, e agora quer buscar os dados do server. Para isso:
git fetch origin
git reset --hard origin/master
